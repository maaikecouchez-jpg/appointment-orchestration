# Fiber – Make an Appointment · Prototype Setup

A responsive, mobile-first prototype of the Telenet **"Maak een afspraak voor je buiteninstallatie"**
(Fiber outdoor-installation appointment) flow, built from the Figma design
*CF – Appointment Orchestration – Playground*.

---

## 1. Goal & scope

Recreate the **full** appointment journey as an interactive prototype:

1. **Intro** – choose *"Eerst beschikbaar"* (first available) or *"Op een ander moment"* (pick your own)
2. **Wizard** (3 steps)
   - Step 1/3 – **Datepicker** + time-slot (moment) selection
   - Step 2/3 – **Extra info** (optional free text)
   - Step 3/3 – **Confirm** appointment (editable summary)
3. **Booked overview** – post-booking dashboard (AppointmentCard, contact, Good to know, FAQ)
4. **Modals & bottom sheets** – info modal ("Outdoor fiber installatie"), edit/reschedule sheets, toasts

Language: **Dutch** (matches the design). Mobile is the source of truth (375px); larger screens
center the experience as a max-width card on the canvas background.

## 2. Tech decisions (confirmed)

| Topic | Decision |
|---|---|
| Framework | **Astro** (static site, responsive) |
| Interactivity | **Vanilla TS islands** – plain `<script>` modules, no UI framework |
| Scope | **Everything** – wizard + overview + modals + sheets |
| Fonts | **Google Fonts lookalikes** (licensed Telenet fonts not bundled) |
| "Today" | **2026-06-11** (current date); first-available computed from data |
| Address & other content | Loaded dynamically from a **variables file** |
| Availability | Mocked **JSON** for the next **4 months**, generated by a script |

### Font mapping (stand-ins)
- **Display / headings** — `PP Right Telenet` → **Archivo** (heavy grotesque, 700/800)
- **Body / UI** — `Telenet Albra Sans` → **Hanken Grotesk** (humanist sans)

(If licensed `.woff2` files are dropped into `public/fonts/` later, swap the `@font-face`
declarations and the CSS variables update everywhere.)

## 4. Data model — mocked API

### `src/data/availability.json`
One entry per **day** for the next 4 months. Each day carries the **4 real spots**;
`Flexibel` is **derived at render time** and never stored.

```jsonc
{
  "generatedAt": "2026-06-11",
  "days": [
    {
      "date": "2026-06-17",
      "available": true,            // true if ≥1 of the 4 spots is available
      "spots": {
        "voormiddag": true,         // morning   08:00–12:00
        "middag":     false,        // noon       10:00–13:30
        "namiddag":   true,         // afternoon  12:30–17:00
        "avond":      false         // evening    17:00–19:00
      }
    }
    // … ~120 days
  ]
}
```

### Moments (UI cards) and their time ranges
| id | Label | Range | Source |
|---|---|---|---|
| `flexibel`   | Flexibel   | 8u – 17u     | **derived**: `voormiddag \|\| middag \|\| namiddag` |
| `voormiddag` | Voormiddag | 8u – 12u     | data |
| `middag`     | Middag     | 10u – 13.30u | data |
| `namiddag`   | Namiddag   | 12.30u – 17u | data |
| `avond`      | Avond      | 17u – 19u    | data |

> Note: `Flexibel` spans the daytime (8–17) and therefore **excludes** `avond` from its rule.

### Generation rules (`scripts/generate-availability.mjs`)
- Range: today (2026-06-11) → +4 months.
- **Weekends** (Sat/Sun) → always unavailable (all spots false).
- **Weekdays** → ~**50%** have availability; the rest fully unavailable.
- For an available weekday → **80%** chance of **≥3** of the 4 spots open, otherwise 1–2 open.
- `available` flag = OR of the 4 spots.
- Re-runnable via `npm run gen:data` (seedable for reproducibility).

- AppointmentCard, contact line, info notice, Good-to-know block, FAQ accordion, AddressInfo

**Modals / sheets / toasts**
- Info modal ("Outdoor fiber installatie" explainer)
- Edit/reschedule bottom sheets, confirmation toast

## 8. Wizard logic (island state machine — `wizard.ts`)

State held in a single object, persisted to `sessionStorage` so steps survive reload:

```ts
type BookingState = {
  mode: "first-available" | "custom";
  date: string | null;          // ISO
  moment: MomentId | null;
  extraInfo: string;
};
```

- **"Eerst beschikbaar"** → prefill `date`/`moment` from `firstAvailable()`, skip the datepicker,
  jump toward confirm.
- **"Op een ander moment"** → full datepicker flow.
- `Verder` disabled until the current step is valid (date+moment chosen, etc.).
- Edit pencils on the confirm screen deep-link back to the relevant step.

## 9. Responsive behaviour

- **≤640px** – full-bleed mobile layout (the Figma design 1:1).
- **>640px** – the 375px experience sits centered as a rounded card on the `--color-canvas`
  background, max-width `--container-max`, with comfortable vertical padding.
- Touch targets ≥44px; grid and moment cards reflow fluidly within the card.
